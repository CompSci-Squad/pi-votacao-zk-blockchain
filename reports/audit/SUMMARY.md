### Slither (executed 2026-05-07)

Run command (with `solc-select` 0.8.24 selected):

```bash
slither src/VotingContract.sol \
    --solc-remaps "@openzeppelin/=lib/openzeppelin-contracts/" \

    --checklist > reports/audit/slither.md
```

Result: **19 findings**, **0 High**, **1 Medium**, **1 Low**, **5 Informational**, **12 naming-convention (Informational)**, **2 Optimization**.

| Severity | Detector | Count | Status |
|---|---|---|---|
| Medium | `reentrancy-no-eth` (cross-function via `nullifiers` mapping) | 1 | **False positive** — function carries `nonReentrant`, and the only external call (`verifier.verifyProof`) is a `view` function on the SnarkJS-generated `PlonkVerifier` (no state, no callbacks). Slither does not consult the `nonReentrant` modifier when computing this detector. CEI ordering is preserved (state writes occur **after** the verify call by design). |
| Low | `reentrancy-benign` (counter increments after external call) | 1 | **False positive** — same reasoning as above; benign counters cannot be exploited because the external call cannot re-enter. |
| Informational | `pragma` (2 different Solidity versions) | 1 | **Expected** — `Verifier.sol` is generated by SnarkJS with `>=0.7.0 <0.9.0`; `VotingContract.sol` pins `0.8.24`. The intersection compiles cleanly under 0.8.24 and is verified by `forge build`. Hand-editing `Verifier.sol` is forbidden by the integration boundary rules. |
| Informational | `cyclomatic-complexity` (`castVote`) | 1 | **Accepted** — `castVote` enforces the full revert-priority chain (`InvalidRaceId → InvalidMerkleRoot → InvalidElectionId → RaceIdMismatch → NullifierAlreadyUsed → InvalidProof`); splitting it would reduce defense-in-depth clarity. |
| Informational | `solc-version` (Slither prefers `0.8.20`+ recommendations) | 1 | **Accepted** — 0.8.24 is intentional (see `foundry.toml`). |
| Informational | `naming-convention` (leading-underscore params) | 12 | Cosmetic; does not affect behavior. Leaving as-is to avoid touching the audited surface. |
| Optimization | `immutable-states` (`admin`, `verifier`) | 2 | Considered. Marking `verifier` `immutable` would forbid the (currently unused) admin path of swapping verifier contracts; `admin` is left mutable to preserve the option for an `Ownable`-style transfer. Acceptable gas trade-off for PoC. |

**Conclusion:** No High-severity findings, no exploitable Medium/Low. The two reentrancy flags are recognized false positives consistent with the `nonReentrant` + view-only-callee design. All Informational items are accepted or expected. Findings are reproducible via the command above.
