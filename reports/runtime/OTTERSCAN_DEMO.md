## Reproducing

```bash
# from repo root
docker compose --profile viz up -d
cd pi-votacao-zk-blockchain
forge build                              # only if out/ is stale
node scripts/leave_vote_for_otterscan.js # casts a fresh vote
```

Each run resets anvil (via `anvil_reset` in the helper) and produces a
fresh tx hash; only the most recent run's hash is valid for navigation.

## Local Sourcify (decoded UI)

The compose stack also runs a Caddy file server at `http://localhost:5102`
that hosts a Sourcify-V1 repo populated by
[`scripts/publish_sourcify.js`](../../scripts/publish_sourcify.js)
immediately after each deploy. Otterscan is configured (via
[`viz/otterscan-config.json`](../../../viz/otterscan-config.json),
mounted over `/usr/share/nginx/html/config.json` in the container) to
treat that URL as a Sourcify source for chainId 31337.

Effect: tx pages now show full function names with parameter names
(e.g. `castVote(uint256 raceId, uint256[24] proof, uint256[5] pubSignals)`)
and decoded event logs (e.g. `VoteCast(uint256 indexed raceId, ...)`)
instead of raw 4-byte selectors and hex blobs. The contract address
pages also expose a "Source" tab with the actual Solidity sources.
